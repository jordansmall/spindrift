# Nix computes; nix-generated bash executes; no third language

> Superseded in part by ADR 0007: the launcher's orchestration logic moves to a
> nix-built Go binary. The tiers below — nix computes, thin generated bash
> executes — still hold for config and the in-container entrypoint; only the
> "no compiled binary" rejection is reversed.

spindrift maximizes pure nix: every decision and constant lives in nix
evaluation, and the runtime residue — the launcher commands and the
in-container entrypoint — is bash *generated by* nix
(`writeShellApplication`), not hand-wired. Config reaches the scripts as a
nix-rendered env preamble derived from the `defaults` attrset, and runtime
tools (`gh`, the container runtime, coreutils) are pinned via `runtimeInputs`,
replacing the hand-maintained `substitute`/`--replace-fail` placeholder
machinery. Bash is confined to the irreducible runtime glue nix cannot
perform: querying GitHub, spawning containers, flipping labels, cloning and
invoking the Agent.

## Considered Options

- **A real CLI in Go/Rust** — more robust runtime code, but adds a language to
  a project whose stated goal is minimizing languages other than nix; rejected.
  Recorded so nobody "upgrades" the launcher to a compiled binary later.
- **Runtime effects in nix itself** (impure derivations, IFD) — fights the
  tool; nix is an evaluation-time language and cannot perform runtime side
  effects; rejected.

## Consequences

`writeShellApplication` shellchecks the generated scripts at build time, and
`runtimeInputs` makes the launchers hermetic (no more hoping the host has
`gh`). The bats test suite deliberately stays bash: nix cannot exercise
runtime behavior, and the fakes-based tests are the right tool for that layer.
